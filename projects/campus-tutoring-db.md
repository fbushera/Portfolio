# Campus Tutoring Center Database

## Overview
A MongoDB database project modeling a campus tutoring center with students, tutors, sessions, bookings, and booking status updates.

## Problem
Tutoring centers need structured data for sessions, tutor availability, student bookings, capacity, location, and booking status changes.

## My Role
I designed and queried MongoDB collections using embedded documents and aggregation pipelines to support tutoring center operations.

## Technologies
- MongoDB
- mongosh
- Embedded documents
- Aggregation pipelines
- Query filters and projections

## Key Features
- Session and booking data modeling
- Queries by capacity, location, student, and booking status
- Aggregation pipeline to analyze bookings
- Nested booking updates using positional operators
- Verification queries after updates

## Skills Demonstrated
- NoSQL data modeling
- MongoDB querying
- Aggregation pipeline design
- Nested document updates
- Database reporting and documentation
